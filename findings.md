# TestPace-Track 发现与研究

## 数据获取方案

### 最终方案: 直接 API 下载

**放弃 Playwright 方案**，原因:
- Playwright chromium 下载慢（约170MB）
- 简化架构，不需要浏览器自动化

**API 信息**:
- URL: `https://onebox.huawei.com/perfect/share/getDocOnlineDownloadUrl{bucket_path}/{doc_id}`
- bucket_path 格式: `/7223826/479248`
- 返回: `{"data": "下载链接", "code": 200}`

## Excel 数据结构

### 合并单元格处理
- 以第一列（特性分类）的合并分布为依据
- 合并行内数据按类型聚合

### 字段类型
| 类型 | 字段示例 | 合并逻辑 |
|------|---------|---------|
| 人员类 | 测试人员、开发人员 | 去重拼接 |
| 是否类 | 是否变更接口 | 有任意"是"则为"是" |
| 数值类 | 用例数、测试进度 | 取非空值或均值 |
| 日期类 | 计划转测时间 | 取第一个非空值 |

---

## 风险判断规则

| 风险类型 | 判断条件 |
|---------|---------|
| 串讲/设计未完成 | 当前时间 > 串讲完成节点 且 串讲进度 ≠ 已完成 |
| 反串讲未完成 | 当前时间 > 反串讲节点 且 反串讲进度 < 100% |
| 测试进度滞后 | 当前时间 > 测试完成节点 且 测试进度 < 100% |
| 空白字段 | 必检字段为空 |

## 需求详情链接

格式: `https://clouddevops.huawei.com/workitem/+{需求编号}`

---

## 必检字段（空白预警）

1. 测试人员
2. 计划转测时间
3. 测试进度
4. 需求编号
5. 需求描述

---

## 日期格式处理

- `2024/4/1` → `2024/04/01`
- `2024.4.1` → `2024/04/01`
- `4/1` → 当前年份 + `/04/01`
- `4.1` → 当前年份 + `/04/01`

---

## 进度格式处理

- "已完成" → `100`
- 百分比数值 → 原值（如 80 → 80）
- `>= 100` 视为完成

---

## 缓存策略

- 位置: `data/cache/`
- 命名: `{文档名称}_{日期}.xlsx`
- 期限: **永久保留**
- 刷新: 按命名覆盖
