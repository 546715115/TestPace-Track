## 一、数据获取配置

### 1.1 下载API的URL

**接口信息：**
- URL: `https://onebox.huawei.com/perfect/share/getDocOnlineDownloadUrl/{云空间编号}/{表格编号}`
- 请求方式: GET
- URL模板: `https://onebox.huawei.com/perfect/share/getDocOnlineDownloadUrl/{bucket_id}/{doc_id}`
- 返回: `{"data": "下载链接", "code": 200}`

**示例：**
- 完整URL: `https://onebox.huawei.com/perfect/share/getDocOnlineDownloadUrl/7223826/479248/31b5087ed7a95f70afe4b5bfbbe215c2`
- 响应中的下载链接: `https://clouddrive-sia.huawei.com/minio/wps-file-bucket/0eb8f92387f840c19e3a849db7246ae7?...`

**可配置参数：**
- 云空间编号（bucket_id）: 如 7223826
- 表格编号（doc_id）: 如 31b5087ed7a95f70afe4b5bfbbe215c2

**需要确认：**
- 是否需要请求头（如 Cookie、Token 等）？
- 配置文件示例：
```json
{
  "documents": [
    {
      "name": "0330需求列表",
      "bucket_id": "7223826",
      "doc_id": "31b5087ed7a95f70afe4b5bfbbe215c2"
    }
  ]
}
```

## 三、风险判断时间节点

### 3.1 串讲/设计完成时间节点

- 时间格式宽松识别: `2024/4/1`, `2024.4.1`, `4.1`, `4/1` 等
- 内部统一转化为 `YYYY/MM/DD` 格式
- 判断逻辑: 当前时间 > 时间节点 且 串讲和测试设计进度 ≠ 已完成 = 有风险

### 3.2 反串讲完成时间节点

- 同3.1时间格式
- 判断逻辑: 当前时间 > 时间节点 且 反串讲进度 < 100% = 有风险
- "已完成"统一转化为 100%

### 3.3 测试转测时间节点

- 同3.1时间格式
- 判断逻辑: 当前时间 > 时间节点 且 测试进度 < 100% = 有风险
- "已完成"统一转化为 100%

## 六、数据缓存策略

**缓存模式：混合模式（本地缓存 + 手动刷新）**

- 缓存存储位置: `data/cache/`
- 缓存过期时间: 一周（7天）
- 点击刷新: 直接覆盖之前的缓存
- 文件命名规则: `{文档名称}_{日期}.xlsx`

**示例：**
```
data/cache/
├── 0330需求列表_20260405.xlsx
├── 0630需求列表_20260405.xlsx
└── ...
```

## 七、页面展示细节

### 7.0 版本计划功能（页面最上方）

**版本计划时间轴功能：**

- 可设置多个阶段（如 Beta_T1、Beta_T2 等）
- 每个阶段包含必选分层：
  - 需求串讲/设计完成阶段
  - 需求反串讲阶段
  - 需求测试阶段
  - 需求测试完成
  - 需求问题单验收完成
- 阶段时间可配置
- 用于动态判断风险

**Beta_T1 阶段示例：**
```
Beta_T1
├── 需求串讲/设计完成: 2024/03/15
├── 需求反串讲完成: 2024/03/20
├── 需求测试完成: 2024/04/01
├── 需求测试完成: 2024/04/10
└── 需求问题单验收完成: 2024/04/15
```
