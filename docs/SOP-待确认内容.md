## 一、数据获取方案（简化版）

### 1.1 方案选择

**选择：直接 API 下载**
- 不使用 Playwright 浏览器自动化
- 直接调用 OneBox API 获取下载链接
- 下载 Excel 后本地缓存分析

### 1.2 下载API的URL

**接口信息：**
- URL: `https://onebox.huawei.com/perfect/share/getDocOnlineDownloadUrl{bucket_path}/{doc_id}`
- 请求方式: GET
- 返回: `{"data": "下载链接", "code": 200}`

**示例：**
- Bucket路径: `/7223826/479248`
- Doc ID: `31b5087ed7a95f70afe4b5bfbbe215c2`
- 完整URL: `https://onebox.huawei.com/perfect/share/getDocOnlineDownloadUrl/7223826/479248/31b5087ed7a95f70afe4b5bfbbe215c2`

**配置文件：**
```json
{
  "documents": [
    {
      "version_id": "0330",
      "name": "0330需求列表",
      "bucket_path": "/7223826/479248",
      "doc_id": "31b5087ed7a95f70afe4b5bfbbe215c2"
    }
  ]
}
```

## 三、风险判断时间节点

### 3.1 串讲/设计完成时间节点

- 时间格式宽松识别: `2024/4/1`, `2024.4.1`, `4.1`, `4/1` 等
- 内部统一转化为 `YYYY/MM/DD` 格式
- 判断逻辑: 当前时间 > 时间节点 且 串讲和测试设计进度 ≠ 已完成 = 有风险

### 3.2 反串讲完成时间节点

- 同3.1时间格式
- 判断逻辑: 当前时间 > 时间节点 且 反串讲进度 < 100% = 有风险
- "已完成"统一转化为 100%

### 3.3 测试转测时间节点

- 同3.1时间格式
- 判断逻辑: 当前时间 > 时间节点 且 测试进度 < 100% = 有风险
- "已完成"统一转化为 100%

## 六、数据缓存策略

**缓存模式：永久本地缓存 + 手动刷新**

- 缓存存储位置: `data/cache/`
- 缓存过期时间: 永久
- 点击刷新: 直接覆盖之前的缓存
- 文件命名规则: `{文档名称}_{日期}.xlsx`

**示例：**
```
data/cache/
├── 0330需求列表_20260405.xlsx
├── 0630需求列表_20260405.xlsx
└── ...
```

## 七、页面展示细节

### 7.0 版本计划功能（页面最上方）

**版本计划时间轴功能：**

- 可设置多个阶段（如 Beta_T1、Beta_T2 等）
- 每个阶段包含必选分层：
  - 需求串讲/设计完成阶段
  - 需求反串讲阶段
  - 需求测试阶段
  - 需求测试完成
  - 需求问题单验收完成
- 阶段时间可配置
- 用于动态判断风险

---

### 7.1 统计仪表盘

展示内容：
- **实际需求数**: X个（需求组合并后）
- **全部需求**: X个（总行数）
- **需求组分解**: X个需求组（包含X行）+ X个单行需求
- 串讲/设计未完成风险数
- 反串讲进度分布
- 测试进度分布

---

### 7.2 风险需求卡片

展示有问题的需求：
- 未开始的需求
- 进度滞后的需求
- 空白字段未填的需求

### 7.3 完整需求列表

- Sheet选择下拉

## 八、风险判断规则汇总

| 风险类型 | 判断条件 |

| 串讲/设计未完成 | 当前时间 > 串讲完成时间节点 且 串讲和测试设计进度 ≠ 已完成 |
