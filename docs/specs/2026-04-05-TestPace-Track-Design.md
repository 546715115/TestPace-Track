**日期**: 2026-04-05
**状态**: 设计中

## 1. 项目概述

### 1.1 项目定位
- **类型**: Web网页工具
- **定位**: 测试进度数据查看器（只读，不操作原表格）
- **面向对象**: 测试人员
- **核心价值**: 将分散的表格数据整合，提供查询、过滤、风险预警功能

### 1.2 数据获取方案（简化版）

### 1.3 数据获取流程

1. 用户在页面上配置文档信息（版本ID、Bucket路径、Doc ID）
2. 点击"下载更新"按钮
3. 后端调用 OneBox API 获取下载链接
4. 下载 Excel 文件到本地缓存
5. 解析 Excel 并展示数据

**API 调用**：
- URL: `https://onebox.huawei.com/perfect/share/getDocOnlineDownloadUrl{bucket_path}/{doc_id}`
- 示例: `https://onebox.huawei.com/perfect/share/getDocOnlineDownloadUrl/7223826/479248/31b5087ed7a95f70afe4b5bfbbe215c2`
- 返回: `{"data": "下载链接", "code": 200}`

## 2. 数据获取模块

### 2.1 API配置（需要网页可配置）

**下载接口**:
- URL模板: `https://onebox.huawei.com/perfect/share/getDocOnlineDownloadUrl{bucket_path}/{doc_id}`
- 请求方式: GET
- 返回: `{"data": "下载链接", "code": 200}`

**配置文件** `config/documents.json`:
```json
{
  "documents": [
    {
      "version_id": "0330",
      "name": "0330需求列表",
      "bucket_path": "/7223826/479248",
      "doc_id": "31b5087ed7a95f70afe4b5bfbbe215c2"
    }
  ]
}
```

### 2.2 缓存策略
- 缓存位置: `data/cache/`
- 文件命名: `{版本名称}_{日期}.xlsx`
- 缓存期限: **永久保留**
- 手动刷新: 按命名覆盖
- 示例:
  - `0330需求列表_20260405.xlsx`
  - `0630需求列表_20260405.xlsx`

## 3. 数据解析模块

### 3.1 合并单元格处理

**合并规则**:
- 以第一列（特性分类）的合并分布为依据
- 合并行内数据按类型聚合:
  - 人员类（测试人员、开发人员等）: 去重拼接，如 "张三,李四"
  - 是否类（是否变更接口等）: 有任意"是"则为"是"，否则为"否"
  - 数值类（用例数、测试进度等）: 取非空值或均值
  - 日期类: 取第一个非空值

**"是否"类字段合并逻辑**:
| 组内值 | 合并结果 |
|--------|---------|
| 全为"否"/"/"/空白 | 否 |
| 有任意"是" | 是 |

### 3.2 字段识别
- 不写死字段名
- 通过Excel表头动态读取
- 检查字段（空白预警）由配置或表头颜色识别

### 3.3 日期格式处理
支持多种输入格式，统一内部存储:
- `2024/4/1` → `2024/04/01`
- `2024.4.1` → `2024/04/01`
- `4/1` → 当前年份 + `/04/01`
- `4.1` → 当前年份 + `/04/01`

### 3.4 进度格式处理
- "已完成" → `100`
- 百分比数值 → 原值（如 80 → 80）
- 判断时: `>= 100` 视为完成

## 4. 版本计划模块

### 4.1 阶段配置

**支持多版本并行**（如 Beta_T1、Beta_T2）

每个版本包含必选分层:

| 分层 | 说明 |
|------|------|
| 需求串讲/设计完成 | 串讲完成时间节点 |
| 需求反串讲完成 | 反串讲完成时间节点 |
| 需求测试阶段 | 测试开始到完成 |
| 需求测试完成 | 测试完成时间节点 |
| 需求问题单验收完成 | 问题单验收完成 |

**数据库表设计**:
```sql
CREATE TABLE version_plan (
    id INTEGER PRIMARY KEY,
    version_id TEXT NOT NULL,
    version_name TEXT NOT NULL,
    stage_name TEXT NOT NULL,
    target_date TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

## 5. 风险判断模块

### 5.1 风险类型

| 风险类型 | 判断条件 |
|---------|---------|
| 串讲/设计未完成 | 当前时间 > 串讲完成节点 且 串讲进度 ≠ 已完成 |
| 反串讲未完成 | 当前时间 > 反串讲节点 且 反串讲进度 < 100% |
| 测试进度滞后 | 当前时间 > 测试完成节点 且 测试进度 < 100% |
| 空白字段 | 必检字段为空 |

### 5.2 风险关联
- 每条需求记录其所属版本计划
- 按版本计划的时间节点判断风险
- 可配置哪些字段需要检查空白

### 6.2 版本选择

每个文档（如"0330需求列表"）对应一个版本。选择版本后：
- 独立的数据缓存
- 独立的版本计划时间轴
- 独立的统计和风险数据

### 6.3 统计仪表盘

| 指标 | 说明 |
|------|------|
| 实际需求数 | 需求组合并后数量 = 需求组数 + 单行需求数 |
| 全部需求 | 总行数 = 需求组内行数 + 单行行数 |
| 需求组分解 | X个需求组（包含Y行）+ X个单行需求 |
| 串讲未完成风险 | 当前时间 > 串讲节点 且 未完成 |
| 反串讲进度分析 | 各进度阶段分布 |
| 测试进度分析 | 各进度阶段分布 |

### 6.4 风险需求卡片
- 按风险类型分组展示
- 点击卡片展开具体需求列表
- 高亮风险字段

### 6.5 完整需求列表

**过滤功能**:
- Sheet选择下拉

- 支持搜索框模糊匹配

**详情链接**:
```
https://clouddevops.huawei.com/workitem/+{需求编号}
```

## 7. 数据库设计

### 7.1 SQLite 表结构

```sql
-- 版本计划表
CREATE TABLE version_plan (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    version_id TEXT NOT NULL,
    version_name TEXT NOT NULL,
    stage_name TEXT NOT NULL,
    target_date TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 文档配置表
CREATE TABLE document_config (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    version_id TEXT NOT NULL UNIQUE,
    name TEXT NOT NULL,
    bucket_path TEXT NOT NULL,
    doc_id TEXT NOT NULL,
    cache_path TEXT,
    last_sync TIMESTAMP,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

## 8. API 设计

### 8.1 后端路由

| 路由 | 方法 | 说明 |
|------|------|------|
| `/` | GET | 主页 |
| `/api/versions` | GET | 获取所有版本配置 |
| `/api/documents` | GET | 获取文档配置列表 |
| `/api/documents` | POST | 创建文档配置 |
| `/api/documents/<version_id>` | PUT | 更新文档配置 |
| `/api/documents/<version_id>` | DELETE | 删除文档配置 |
| `/api/sheets` | GET | 获取所有Sheet页 |
| `/api/load_sheet` | POST | 加载指定Sheet数据 |
| `/api/download` | POST | 下载并缓存Excel |
| `/api/version_plans` | GET | 获取版本计划列表 |
| `/api/version_plans` | POST | 新增版本计划 |
| `/api/version_plans/<id>` | PUT | 更新版本计划 |
| `/api/version_plans/<id>` | DELETE | 删除版本计划 |
| `/api/stats` | GET | 获取统计数据 |
| `/api/risks` | GET | 获取风险列表 |

### 8.2 响应格式

```json
{
  "success": true,
  "data": { ... },
  "error": null
}
```
